# 01-product.md — 제품 정의

## 제품 목표

> **팀 업무를 시각화한다. "지금 누가 뭐 해?" 라는 질문이 사라지게 만든다.**

누군가에게 물어봐야만 현황을 알 수 있는 팀은 소통 비용이 높다.
TaskFlow2는 업무 상태를 항상 화면에서 확인할 수 있게 해 그 비용을 제거한다.

---

## 페르소나

| 항목 | 내용 |
|------|------|
| **대상** | 10명 규모 스타트업의 팀 리더 |
| **연령대** | 30~40대 |
| **주요 고통** | 슬랙·메신저로 진행 상황을 일일이 물어봐야 함 |
| **원하는 것** | 지금 누가 무엇을 하는지, 언제 끝나는지 한눈에 보고 싶다 |
| **기술 수준** | 웹 앱 사용에 익숙하나 개발자는 아님 |

## MVP 범위

### 핵심 기능 — CRUD 4종 전부 화면에서 동작

| 동작 | 설명 |
|------|------|
| **추가 (Create)** | 새 업무 생성 폼 (제목, 상태, 마감 시각) |
| **목록 (Read)** | 전체 업무 목록 조회, 상태별 필터 |
| **수정 (Update)** | 인라인 또는 모달 편집 |
| **삭제 (Delete)** | 확인 후 삭제 |

### 업무 속성

- **상태 분류**: `todo` / `in_progress` / `done`
- **마감 시각 (due_at)**: 날짜 + 시간까지 지정
  - 형식: `YYYY-MM-DD HH:mm` (예: `2026-05-12 18:00`)
  - 날짜만 저장하지 않는다. 시간 단위 마감 관리가 목적이다.

### UI/UX

- **테마 토글**: 라이트 / 다크 모드 전환, `localStorage`에 저장하여 새로고침 후에도 유지
- **반응형**: 최소 너비 360px에서 레이아웃 깨짐 없음 (모바일 우선)
- **디자인 톤**: macOS 스타일
  - 둥근 모서리 (border-radius 기반)
  - 부드러운 그림자 (box-shadow, 강하지 않게)
  - 반투명 카드 (backdrop-filter 또는 rgba 배경)
  - 시스템 폰트 (`-apple-system, BlinkMacSystemFont, "Segoe UI", sans-serif`)

## 확장 계획 (MVP 이후 단계)

> 아래 항목은 설계 시 고려하되, **MVP에서는 구현하지 않는다.**

| 기능 | 단계 |
|------|------|
| JWT 로그인 / 인증 | 확장 1단계 |
| 팀 단위 업무 분리 | 확장 1단계 |
| Kanban 보드 뷰 | 확장 2단계 |
| 팀 채팅 | 확장 3단계 |
| CI/CD 파이프라인 | 확장 2단계 |

## MVP 범위 외 (명시적 제외)

| 항목 | 이유 |
|------|------|
| 외부 결제 | MVP 수익 모델 없음 |
| 네이티브 앱 (iOS/Android) | 반응형 웹으로 대체 |
| WebRTC (영상/음성) | 범위 초과 |
| 외부 캘린더 연동 | 복잡도 과도 |
| 파일 업로드 | 스토리지 인프라 불필요 |
| WebSocket 실시간 | MVP는 **폴링**으로 대체, WebSocket은 확장 단계에서 도입 |

## 성공 기준

MVP가 완료되었다고 판단하는 조건. **전부 통과해야 완료다.**

| 기준 | 측정 방법 |
|------|-----------|
| 새로고침 후 테마 유지 | `localStorage` 값 확인, 화면 재진입 테스트 |
| 360px에서 레이아웃 깨짐 없음 | 브라우저 DevTools 360px 뷰포트 확인 |
| API 응답 200ms 이하 (CRUD 4종) | 네트워크 탭 또는 서버 로그 측정 |
| CRUD 4종 화면에서 정상 동작 | 추가 → 수정 → 삭제 → 목록 확인 시나리오 통과 |
| 테마 토글 정상 동작 | 라이트 ↔ 다크 전환 시 전체 UI 적용 확인 |
