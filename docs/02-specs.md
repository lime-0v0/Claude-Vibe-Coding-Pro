# 02-specs.md — 기술 명세

## Task 데이터 모델

### 필드 정의

| 필드 | 타입 | 필수 | 기본값 | 설명 |
|------|------|------|--------|------|
| `id` | INTEGER (PK, AUTO) | — | auto | 고유 식별자 |
| `title` | VARCHAR(200) | **필수** | — | 업무 제목 |
| `description` | TEXT | 선택 | `null` | 업무 상세 설명 |
| `status` | ENUM | **필수** | `todo` | 진행 상태 (`todo` / `in_progress` / `done`) |
| `due_at` | DATETIME (UTC) | 선택 | `null` | 마감 시각, ISO 8601 형식으로 저장 |
| `created_at` | DATETIME (UTC) | — | auto | 생성 시각, 서버 자동 기록 |
| `updated_at` | DATETIME (UTC) | — | auto | 수정 시각, 서버 자동 갱신 |

### 상태(status) 전이

```
todo  →  in_progress  →  done
 ↑____________↑____________↑   (역방향도 허용)
```

---

## 입력 검증 규칙

| 조건 | HTTP 응답 | 설명 |
|------|-----------|------|
| `title` 누락 또는 빈 문자열 | `400 Bad Request` | 필수 필드 |
| `title` 200자 초과 | `400 Bad Request` | 길이 제한 |
| `status` 허용 값 외 | `400 Bad Request` | `todo`, `in_progress`, `done` 만 허용 |
| `due_at` ISO 8601 형식 위반 | `400 Bad Request` | 예: `2026-05-12T18:00:00Z` 형식만 허용 |
| 존재하지 않는 `id` 요청 | `404 Not Found` | GET 단건, PUT, DELETE 공통 |

**ISO 8601 허용 예시**
```
2026-05-12T18:00:00Z        # UTC 명시
2026-05-12T18:00:00+09:00   # 타임존 오프셋
```

## REST API 명세

**Base URL**: `/api/tasks`

### 1. 업무 추가

```
POST /api/tasks
```

| 항목 | 내용 |
|------|------|
| **요청 바디** | `title` (필수), `description` (선택), `status` (선택), `due_at` (선택) |
| **응답 코드** | `201 Created` |
| **응답 바디** | 생성된 Task 단건 (전체 필드 포함) |

```json
// 요청 예시
{
  "title": "디자인 시스템 정의",
  "description": "컬러, 타이포, 컴포넌트 토큰 정의",
  "status": "todo",
  "due_at": "2026-05-12T18:00:00Z"
}
```

---

### 2. 업무 목록 조회

```
GET /api/tasks
```

| 항목 | 내용 |
|------|------|
| **쿼리 파라미터** | `status` (선택, 필터용) |
| **응답 코드** | `200 OK` |
| **응답 바디** | Task 배열 — `description` **제외** |

```json
// 응답 예시
[
  {
    "id": 1,
    "title": "디자인 시스템 정의",
    "status": "todo",
    "due_at": "2026-05-12T18:00:00Z",
    "created_at": "2026-05-01T09:00:00Z",
    "updated_at": "2026-05-01T09:00:00Z"
  }
]
```

> `description` 은 목록에서 제외한다. 카드 렌더링 비용을 줄이기 위함이다.

---

### 3. 업무 단건 조회

```
GET /api/tasks/:id
```

| 항목 | 내용 |
|------|------|
| **응답 코드** | `200 OK` |
| **응답 바디** | Task 단건 — `description` **포함** (전체 필드) |
| **오류** | 없는 id → `404 Not Found` |

---

### 4. 업무 수정

```
PUT /api/tasks/:id
```

| 항목 | 내용 |
|------|------|
| **요청 바디** | 수정할 필드만 전송 (부분 수정 허용) |
| **응답 코드** | `200 OK` |
| **응답 바디** | 수정된 Task 단건 (전체 필드 포함) |
| **오류** | 없는 id → `404` / 검증 실패 → `400` |

```json
// 부분 수정 예시 — status 만 변경
{ "status": "in_progress" }
```

---

### 5. 업무 삭제

```
DELETE /api/tasks/:id
```

| 항목 | 내용 |
|------|------|
| **응답 코드** | `204 No Content` |
| **응답 바디** | 없음 |
| **오류** | 없는 id → `404 Not Found` |

## 화면 명세 — CRUD 4종 UI

### 추가 화면 — 업무 생성 폼

```
┌─────────────────────────────────┐
│  새 업무 추가                    │
│                                 │
│  제목 *                         │
│  [ 업무 제목을 입력하세요       ] │
│                                 │
│  마감 시각                       │
│  [ 2026-05-12  18:00           ] │
│                                 │
│  상태                            │
│  ( todo ) ( in_progress ) ( done)│
│                                 │
│              [ 취소 ] [ 추가 ✓ ] │
└─────────────────────────────────┘
```

- `title` 은 필수, 나머지는 선택
- `due_at` 입력은 날짜 + 시간 picker (분 단위)
- 제출 시 `POST /api/tasks` → 성공 시 목록에 즉시 추가

---

### 목록 화면 — 업무 카드

```
┌───────────────────────────────────┐
│  [todo] 디자인 시스템 정의    🗑   │
│         D-3  18:00                │
└───────────────────────────────────┘
┌───────────────────────────────────┐
│  [in_progress] API 명세 작성  🗑   │
│         D-0  09:00                │
└───────────────────────────────────┘
```

- **status 배지**: `todo` / `in_progress` / `done` 색상 구분
- **D-N HH:MM**: `due_at` 기준 오늘 대비 남은 일수 + 마감 시각
  - 당일: `D-0 18:00`
  - 초과: `D+2 18:00` (붉은색)
  - `due_at` 없음: 표시 생략
- 카드 클릭 → 수정 모달 진입

---

### 수정 화면 — 카드 클릭 → 모달

```
┌─────────────────────────────────┐
│  업무 수정                  ✕   │
│                                 │
│  제목 *                         │
│  [ 디자인 시스템 정의          ] │
│                                 │
│  설명                            │
│  [ 컬러, 타이포, 컴포넌트 토큰  ] │
│                                 │
│  마감 시각                       │
│  [ 2026-05-12  18:00           ] │
│                                 │
│  상태                            │
│  ( todo ) (●in_progress) ( done )│
│                                 │
│              [ 취소 ] [ 저장 ✓ ] │
└─────────────────────────────────┘
```

- 모달 진입 시 `GET /api/tasks/:id` 호출 → `description` 포함 전체 필드 표시
- 저장 시 `PUT /api/tasks/:id` (변경된 필드만 전송)
- 성공 시 모달 닫힘 + 목록 카드 즉시 갱신

---

### 삭제 흐름 — 휴지통 → 확인 → DELETE

```
1. 카드 우측 🗑 아이콘 클릭

2. 확인 다이얼로그 표시
   ┌───────────────────────────────┐
   │  정말 삭제하시겠습니까?        │
   │  "디자인 시스템 정의"          │
   │                               │
   │          [ 취소 ] [ 삭제 ]    │
   └───────────────────────────────┘

3. [삭제] 클릭 → DELETE /api/tasks/:id
4. 204 응답 → 목록에서 카드 제거 (애니메이션)
```

- 🗑 클릭이 카드 클릭(수정 모달)과 겹치지 않도록 이벤트 전파 차단(`stopPropagation`)
