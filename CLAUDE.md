## 절대 규칙 (위반 불가)

### 1. 추측 금지
확실하지 않은 정보를 추측하여 코드를 작성하지 않는다.
요구사항이 불명확하면 반드시 먼저 확인 후 진행한다.

### 2. 돌발 의존성 금지
새 라이브러리·패키지·외부 서비스를 사용자 승인 없이 임의로 추가하지 않는다.
필요 시 이유와 대안을 제시하고 승인을 받는다.

### 3. 테스트 없이 완료 금지
모든 기능 구현은 단위 테스트 또는 통합 테스트를 포함해야 완료로 간주한다.
테스트 작성이 불가능한 경우 그 이유를 명시한다.

### 4. 시크릿 하드코딩 금지
API 키, 비밀번호, 토큰, DB 접속 정보 등 민감 정보는 절대 코드에 직접 작성하지 않는다.
반드시 환경 변수(`.env`) 또는 시크릿 관리 도구를 사용한다.

### 5. 폴더 구조 임의 변경 금지
`docs/05-conventions.md`에 정의된 폴더 구조를 무단으로 변경하지 않는다.
구조 변경이 필요하면 사용자에게 이유를 설명하고 승인을 받는다.

## 모호한 요청 처리 원칙

요청이 불명확하거나 해석이 2가지 이상 가능한 경우, **추측하지 않고** 선택지 2~3개를 제시하여 되묻는다.

**예시 형식:**
```
요청이 다음 중 어느 방향인지 확인이 필요합니다.

1. [해석 A] — 간략한 설명
2. [해석 B] — 간략한 설명
3. [해석 C] — 간략한 설명 (해당하는 경우)

어떤 방향으로 진행할까요?
```
